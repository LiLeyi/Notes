    继承和组合

    继承目的：提供代码重用可能性

    class A {};
class X: A {
    // ...
};

// C++也可以继承多个类

class B {};
class X: A, B {
    // ...
}

    class X {
    public:
    X(int a) {};
};

class Y:public X {
    X x;
    public:
    // 初始化应该这样写
    Y(int a, int b): X(a), x(b) {};
};

    访问权限问题：

    • 父类中private子类无法访问

    对子类初始化时，会先触发父类构造函数，再触发子类构造函数

    多继承时，例如继承BaseA，BaseB，BaseC，则先初始化BaseA，再初始化BaseB，最后初始化BaseC。然后再按顺序创建数据成员

    #include <iostream>

class BaseA
{
    public:
    BaseA()
    {
        std::cout << "BaseA constructor" << std::endl;
    }
};

class BaseB
{
    public:
    BaseB()
    {
        std::cout << "BaseB constructor" << std::endl;
    }
};

class BaseC
{
    public:
    BaseC()
    {
        std::cout << "BaseC constructor" << std::endl;
    }
};

class MemberX
{
    public:
    MemberX()
    {
        std::cout << "MemberX constructor" << std::endl;
    }
};

class MemberY
{
    public:
    MemberY()
    {
        std::cout << "MemberY constructor" << std::endl;
    }
};

class Derived: public BaseC, public BaseB, public BaseA
{
    public:
    MemberX mx;
    MemberY my;
    Derived()
    {
        std::cout << "Derived constructor" << std::endl;
    }
};

int main()
{
    Derived d;
    return 0;
}

/*
    BaseC constructor
    BaseB constructor
    BaseA constructor
    MemberX constructor
    MemberY constructor
    Derived constructor
*/

    所以构造基类的顺序看继承那里，而不是构造函数中传递参数的顺序

    子类和父类有方法名字冲突时，默认用子类的。加命名空间后可以使用父类的，例如d.类名::函数名

    虚基类：

    class B: virtual public A {}

    说明A在继承链路中，若出现多个A，则只继承一个

    例如：

    struct A {int m;}
struct B1: virtual public A {};
struct B2: virtual public A {};
struct B3: public A;
struct B4: public A;

struct C: public B1, public B2, public B3, public B4 {}
// 则C中有3个A，前两个是虚继承，只创建一个，后面没virtual，则创建两个

    构造、析构、=运算符不能继承。

    静态成员函数不能是虚函数，没有地址指针，只能够访问静态成员

    继承前加private：

    • 父类public成员变为private
    • 父类protected成员变为private
    • 父类private变为不可访问

    继承前加protected：

    • 父类public成员变为protected
    • 父类protected成员变为protected
    • 父类private变为不可访问

    用class默认为私有，用struct则默认为public

    赋值兼容性规则：

    • 可以使用派生类来初始化基类对象
    • 将派生类对象指针赋值给基类指针
    class Base{
    int m;
}
class Derived: public Base{}

int main() {
    Base b;
    Derived d(20);
    b = d; // 只赋值了基类的东西，派生类多的会被切片
}

    派生类地址可以复制给基类指针（反过来不行）

    虚函数

    通过函数重载实现，基类和派生类完全相同的函数声明

    class Shape
{
    void disp()
    {
        cout << "this is shape" << endl;
    }
};

class Triangle
{
    void disp()
    {
        cout << "this is triangle" << endl;
    }
}

int main()
{
    Triangle t;
    Shape * s = t;
    s.disp(); // 输出 this is shape，如果想要输出triangle，就需要虚函数
}

    class Shape
{
    virtual void disp() = 0; // 只有virtual才能写=0，意味着该函数只能做虚函数，派生类必须实现纯虚函数（可以不写=0）。有纯虚函数的类称为抽象类，无法初始化。虚函数不能是静态的
};

class Triangle
{
    void disp()
    {
        cout << "this is triangle" << endl;
    }
}

int main()
{
    Triangle t;
    Shape * s = t;
    s.disp(); // 输出 this is triangle。要是有多重继承，触发最远端的函数
}

    编译器创建VTABLE专门存储有虚函数的类，调用时查表

    纯抽象类：所以函数都是纯虚函数

    构造函数不能是虚函数，但析构函数可以是虚函数

    void main()
{
    A *a = new B(15);
    delete a;
}
/*
    无虚析构函数：
    A::A()
    B::B(int)
    A::~A()

    有虚析构函数：
    A::A()
    B::B(int)
    B::~B()
    A::~A()
*/

    析构触发顺序：自身、数据成员、基类

    运算符重载也可以添加virtual
