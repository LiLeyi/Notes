    SQL

    结构化查询语言（Structured Query Language）

    模式由基本表构成，SQL包含视图和基本表

    创建

    • 模式：CREATE SCHEMA。CREATE SCHEMA <模式名（可选参数，默认为用户名）> AUTHORIZATION <用户名>
    • 表：CREATE TABLE
    • 视图：CREATE VIEW
    • 索引：CREATE INDEX

    删除：

    • 模式：DROP SCHEMA
    • 表：DROP TABLE
    • 视图：DROP VIEW
    • 索引：DROP INDEX

    修改：

    • 表；ALTER TABLE

    模式

    CREATE SCHEMA <模式名（可选参数，默认为用户名）> AUTHORIZATION <用户名>

    如果现在登录的用户是Sue，Select * from mytable，搜索sys.mytable，然后是Sue.mytable，最后搜寻其他用户.mytable

    每个数据库创建后，有四个schema无法删除，dbo、guest、sys、INFORMATION_SCHEMA

    CREATE SCHEMA ＜模式名＞ AUTHORIZATION ＜用户名＞［＜表定义子句＞｜＜视图定义子句＞｜＜授权定义子句＞］

    例创建相互依赖的 FOREIGN KEY 约束 

    CREATE SCHEMA AUTHORIZATION ross 
               CREATE TABLE t1 (c1 INT PRIMARY KEY, c2 INT REFERENCES t2(c1))
               CREATE TABLE t2 (c1 INT PRIMARY KEY, c2 INT REFERENCES t1(c1))

    DROP SCHEMA ＜模式名＞ [CASCADE | RESTRICT]

    • CASCADE级联式：删除模式下的所有数据库对象
    • RESTRICT限制式：表示如果该模式中已经定义了下属的数据库对象，则拒绝该删除语句的执行

    表

    定义基本表

    CREATE TABLE <表名> (
    <列名><数据类型>[<列级完整性约束>]
    [,<列名><数据类型>[<列级完整性约束>]] ...
    [,<表级完整性约束>] ...
)

    表级完整性约束：有时两个列间互相有约束，这时需要使用表级约束（例如a依赖b，b依赖a，就要写在这里面）

    使用CONSTRAINT子句定义列级或表级约束。格式为

    CONSTRAINT <约束名> <约束>

    例如

    CREATE TABLE Student(
    Sno CHAR(6) NOT NULL UNIQUE,
    Sname Char(8),
    Sage INT,
    Ssex CHAR(2)
    Sdept CHAR(12),
    CONSTRAINT C1 CHECK (Ssex IN ('男', '女')),
    CONSTRAINT S_PK PRIMARY KEY(Sno)
);

    对于PRIMARY KEY和FOREIGN KEY，可以省略CONSTRAINT以及约束名。例如

    FOREIGN KEY(Sno) REFERENCES Student(Sno)

    如果有多个表的属性域是相同的，通过对域的修改可以很容易地改变属性的数据类型

    CREATE DOMAIN <域名> <数据类型>

    创建域后，类型定义可以使用该域名

    修改基本表

    ALTER TABLE <表名>
    [ADD <列名><数据类型>[<列级完整性约束>]]
    [DROP <列名> [CASCADE | RESTRICT]]
    [ALTER <列名> <数据类型>]
    • CASCADE级联式：删除模式下的所有数据库对象。引用该列的表、视图等都会被删掉
    • RESTRICT限制式（默认）：表示如果已有其他对象引用此列，则拒绝该删除语句的执行

    ADD的新列全为空值

    修改原有列定义有可能会破坏已有数据

    DROP TABLE ＜表名＞ [RESTRICT｜CASCADE];

    对SQL Server，无RESTRICT和CASCADE选项。数据、表上的索引都删除,删除基本表时，系统会从数据字典中删去有关该基本表及其索引的描述。表上的视图往往仍然保留，但无法引用。DROP TABLE 不能用于删除有 FOREIGN KEY 约束引用的表

    Oracle删除表时,自动删除表上所有索引,保留视图定义(但失效),保留建立在此表上的存储过程和函数(但失效), CASCADE选项下可以删除有 FOREIGN KEY 约束引用的表

    索引

    加快查询速度。可在一个表上建立一个、多个索引，以提供多种存取路径，加快查找速度

    建立索引：DBQ或表的属主（建表人）根据需要建立。有些DBMS自动建立以下列上的索引（PRIMARY KEY，UNIQUE）

    维护索引：DBMS自动完成

    使用索引：系统在存取数据时会自动选择是否使用索引以或合适的索引作为存取路径，用户不必也不能选择索引

    CREATE [UNIQUE] [CLUSTER] INDEX <索引名> ON <表名> (
    <列名> [次序]
    [,<列名> [次序]] ...
)

    次序指索引值排列次序，升序ASC（默认），降序DESC

    • UNIQUE表明每一个索引只对应唯一的数据记录。对于已经含有重复值属性列不能建UNIQUE索引。插入新纪录时，DBMS会自动检查新纪录在该列上是否取了新值
    • CLUSTER表示要建立的索引是聚簇索引。叶节点中存储索引和数据，非聚簇是分开的。一个表只能有一个聚簇索引。即将一行数据存在一起。适用于很少对表进行增删操作，很少对其中的变长列（例如VARCHAR）进行修改操作。对经常搜索范围值的列特别有效
    DROP INDEX <索引名>

    无CASCADE和RESTRICT的区别

    索引会带来额外内存，对数据插入、更新、删除花费时间更长。需要估计在数据库上用什么样的查询组合以及其他操作

    数据操作

    查询

    单表查询

    SELECT [ALL|DISTINCT] <目标表达式> [, <目标表达式>] ...
    FROM <表名或视图名> [, 表名或视图名] ...
    [WHERE <条件表达式>]
    [GROUP BY <列名1> [HAVING <条件表达式>]]
    [ORDER BY <列名> [ASC|DESC]]

    因为查询结果中删除重复元组很复杂，所以数据库中一般允许在关系和SQL中出现重复元组。但使用DISTINCT就会进行去重

    写SELECT * FROM ...，则选择表中所有列

    目标表达式可以是列名、算术表达式 、字符串常量、函数、列别名（用AS改名字）等

    单表查询：

    SELECT Sname AS NAME, 'Year of Birth:', 2000-Sage, LOWER(Sdept) FROM STUDENT
-- 输出：
-- NAME 'Year of Birth' 2000-Sage LOWER(Sdept)
-- 李勇 'Year of Birth' 1997       cs
    • 比较表达式：<列名1> 比较算符 <列名2（或常量）>。比较算符有=、>、<、<=、>=、<>（或!=）
    • 逻辑表达式：<条件表达式1> 逻辑算符 <条件表达式2>。逻辑算符有AND、OR、NOT
    • BETWEEN：<列名> (NOT) BETWEEN <常量1或列名2> AND <常量2或列名3>
    • IN：<列名> (NOT) IN （常量表列或SELECT语句）
    • LIKE：<列名> (NOT) LIKE '<匹配字符串>' [ESCAPE '<换码字符>']。匹配串中，_表示匹配一个字符，%表示匹配任意字符（长度可以为0）。例如'a%b'表示开头为a，结尾为b，'a_b'表示开头为a，结尾为b，且中间还有一个字符。若查询的字符串本来就有%或_，则需要ESCAPE对通配符转义，例如WHERE Cname LIKE 'DB\_Design' ESCAPE '\'
    • NULL：<列名> IS (NOT) NULL
    • EXISTS：(NOT) EXISTS (SELECT语句)
    • IN：(NOT) IN <值表>。WHERE Sdept NOT IN ('IS', 'MA', 'CS')

    空值和空白、零值不一样。空值和任何值不相等，与任何值进行算术运算，结果仍然空值。执行计算时消除空值很重要。使用逻辑、比较运算符，有可能返回结果UNKNOWN，是一个布尔值

    空串为零长度字符串

    AND优先级大于OR，可以使用括号改变优先级

    ORDER BY 列名 [ASC|DESC]。对于空值，大部分数据库系统将其作为最大值排序

    集函数

    主要集函数：

    • COUNT(*)
    • COUNT([DISTINCT|ALL] <列名>)
    • SUM([DISTINCT|ALL] <列名>)
    • AVG([DISTINCT|ALL] <列名>)
    • MAX([DISTINCT|ALL] <列名>)
    • MIN([DISTINCT|ALL] <列名>)

    ALL为默认值

    SELECT COUNT(*) FROM STUDENT;

SELECT COUNT(DISTINCT Sno) FROM SC;

SELECT AVG(Grade) FROM SC WHERE Cno='1';

    集函数可以用GROUP BY分组，细化集函数作用对象。分组后集函数分别作用于每个组。分组方法：按指定的一列或多列值分组，值相等的为一组

    -- 求每个课程号及相应的选课人数
SELECT Cno, COUNT(Sno) FROM SC GROUP BY Cno;

    使用GROUP BY子句后，SELECT子句的列名列表中只能出现分组属性和集函数

    -- 错误
SELECT Cno, COUNT(Sno), Grade FROM SC GROUP BY Cno;

    GROUP BY作用对象是查询的中间结果表，使用HAVING短语指定条件的组才输出

    -- 查询选修了3门以上课程的学生学号
SELECT Sno FROM SC 
    GROUP BY Sno
    HAVING COUNT(*) > 3;

    Where子句作用于基表或视图，从中选择满足条件的元组。其中不能用聚集函数

    HAVING作用于组，从中选择满足条件的组。HAVING前一定有GROUP BY

    连接查询

    • 广义笛卡尔积
    • 等值（含自然连接）与非等值连接查询
    • 自身连接查询
    • 外连接查询
    • 符合条件连接查询

    广义笛卡尔积

    不带连接谓词的连接（没有WHERE）。很少用，因为会产生无意义元组

    SELECT Student.*, SC.* FROM Student, SC;

    等值与非等值连接查询

    等值连接

    SELECT Student.*, SC.*
    FROM Student, SC
    WHERE Student.Sno = SC.Sno;

    连接查询

    SELECT buyer_name, sales.buyer_id, qty
    FROM sales, buyer INNER JOIN buyer
    ON buyer.buyer_id = sales.buyer_id;

    自身连接：查询先修课的先修课

    SELECT FIRST.Cno, SECOND.Cpno
    FROM Course FIRST, Course SECOND
    WHERE FIRST.Cno = SECOND.Cpno;

    第二行其实是别名

    外连接：

    SELECT Student.Sno,Sname,Ssex, Sage,Sdept,Cno,Grade
    FROM Student LEFT JOIN SC ON (Student.Sno = SC.Sno);

    嵌套查询：一个SELECT FROM WHERE语句称为一个查询快。将一个查询块嵌套在另一个查询块的WHERE子句或HAVING短语的条件中的查询称为嵌套查询。子查询不能用ORDER BY

    SELECT Sname
    FROM Student
    WHERE Sno IN
        (SELECT Sno FROM SC WHERE Cno = '2')

    实现一般是从里到外，先进行子查询，再把其结果用于父查询

    相关子查询：子查询的查询条件依赖于父查询。遍历外层，对每行，外层向内层子查询传递数据

    不相关子查询从内到外逐层处理

    引出子查询：

    • IN

    查询刘晨在同一个系学习的学生

    1. 确定刘晨所在系名
    SELECT Sdept FROM Student WHERE Sname = '刘晨'
-- 输出：
-- Sdept
-- cs

    2.查找所有在同一系学习的学生

    SELECT Sno, Sname, Sdep
  FROM Student
  WHERE Sdept IN (
      SELECT Sdept
          FROM Student
          WHERE Sname = '刘晨'
  )

    为不相关子查询。先算内层再算外层

    • WHERE
    • 带ANY或ALL谓词的子查询。需要配合比较运算符。例如>ANY

    例：查询其他系中比CS系所有学生年龄小的学生姓名和年龄

    SELECT Sname, Sage FROM Student Where Sage <ALL (SELECT Sage FROM Student WHERE Sdept = 'CS') AND Sdept <> 'CS';

SELECT Sname, Sage FROM Student Where Sage < (SELECT MIN(Sage) FROM Student WHERE Sdept = 'CS') AND Sdept <> 'CS';

    对应关系

    = <>或!- < <= > >=
    ANY IN - <MAX <=MAX >MIN >=MIN
    ALL - NOT IN <MIN <= MIN >MAX >= MAX

    通常用集函数实现子查询比直接用ANY或ALL查询效率要高，前者能减少比较次数

    EXISTS谓词，带有该谓词的子查询不返回数据，只产生true或false，即如果内层查询结果非空，则返回true，否则返回false。其SELECT的目标列通常用*代替，因为此时选择目标列无意义

    NOT EXISTS...

    例：查询所有选修了1号课程的学生姓名

    SELECT Sname FROM Student 
  WHERE EXISTS (
      SELECT * FROM SC
          WHERE Sno = Student.Sno AND Cno = "1"
  );

    为相关子查询，因为子查询用到了父查询的属性

    不同形式查询间的转换：

    • 有些带EXISTS、NOT EXISTS的子查询不能被其他形式的子查询替换
    • 所有带IN、比较运算符、ANY、ALL谓词的子查询都能用带EXISTS的子查询等价替换
    • 带EXISTS谓词的相关子查询只关心内层查询是否有返回值，不需要查具体值，效率不低于相关子查询

    例：查询与刘晨在同一个系学习的学生

    SELECT * FROM Student
  WHERE EXISTS (
      SELECT * FROM Student S2
          WHERE S2.Sdept = S1.Sdept AND S2.name = '刘晨'   
);

    会将所有判断条件下推

    使用EXISTS/NOT EXISTS实现全称量词（SQL中没有全称量词）

    \[(\forall x) P \equiv \neg (\exists x (\neg P))\]

    例：查询选修了全部课程的学生姓名

    SELECT Sname FROM Student
  WHERE NOT EXISTS (
    SELECT * FROM Course
      WHERE NOT EXISTS (
          SELECT * FROM SC 
              WHERE Sno = Student.Sno And Cno = Course.Cno
  )
)

    实现蕴含逻辑

    \[p\rightarrow q \equiv \neg p \lor q\]

    例：查询至少选修了学生22选修的全部课程的学生号码

    查询学号为x的学生，对所有课程y，只要22学生选修了课程y，则x也选修了y。形式化表述：

    • p：22选修了课程y
    • q：x选修了课程y
    • 则\((\forall y) p \rightarrow q\)，等价变换为\(\neg \exists y(p\land \neg q)\)

    即语义：不存在课程y，学生22选修了y，而学生x没有选

    sql SELECT Sno FROM Student WHERE NOT EXISTS ( SELECT * FROM COURSE WHERE ......//后面再看看 )

    数据更新

    两种插入数据方式

    • 插入单个元组。要保证VALUES中和表的属性对应
    INSERT INTO <表名>[(<属性列1>[, <属性列2>]...)] VALUES([值1], [值2], ...);

INSERT INTO Student VALUES('200715128', '李', 19);
    • 插入子查询结果
    INSERT INTO <表名>[<列名1>[, <列名2>]...] <SELECT语句>

INSERT INTO Dept_age(Sdept, Avg_age)
    SELECT Sdept, AVG(Sage)
        FROM Student
        GROUP BY Sdept;

    插入时会检查所插元组是否破坏表上已定义的完整规则

    • 实体完整性
    • 参照完整性
    • 用户自定义完整性
    • NOT NULL检查是否非空
    • UNIQUE检查是否有重复值
    • 对有值域约束的属性列所提供的属性值是否在值域范围内

    修改数据

    UPDATE <表名>
    SET 列名1=<表达式1>[, 列名2=<表达式2>]...
    [WHERE <条件表达式>]

    修改方式：

    • 单个元组修改
    UPDATE Student SET Sage=22 WHERE Sno = '22';
    • 修改多个元组的值
    UPDATE Student SET Sage=Sage+1;

UPDATE Student SET Sage=Sage+1 WHERE Sdept='cs';
    • 带子查询的修改语句
    UPDATE SC SET Grade=0
    WHERE 'CS' = (
        SELECT Sdept FROM Student
            WHERE Student.Sno = SC.Sno
    );

    修改时也会检查完整性。注意：

    • 主键不允许被修改

    删除数据

    DELETE FROM <表名> [WHERE <条件>]

    三种删除方式：

    • 删除一个元组
    DELETE FROM Student WHERE Sno='11';
    • 删除多个元组的值
    DELETE FROM SC WHERE Sno='2'

DELETE FROM SC
    • 带子查询的删除语句
    DELETE FROM SC
    WHERE  'CS'=
        (SELECT Sdept 
            FROM Student
            WHERE Student.Sno=SC.Sno)

    删除表中的所有行，而不记录单个行删除操作

    TRUNCATE TABLE table_name
    • 功能上与不带 WHERE 子句的 DELETE 语句相同：二者均删除表中的全部行。
    • 但 TRUNCATE TABLE 比 DELETE 速度快，且使用的系统和事务日志资源少。
    • DELETE 语句每次删除一行，并在事务日志中为所删除的每行记录一项。
    • TRUNCATE TABLE 通过释放存储表数据所用的数据页来删除数据，并且只在事务日志中记录页的释放。
    • 操作不能回滚， DELETE 可以回滚

    DBMS在执行增删改语句时必须保证数据库一致性。数据库不一致性的形成：

    • 当某同学退学时，首先删除成绩表（子表）中数据，然后删除学籍表（主表）中的数据，但如果执行完第一步后，计算机发生故障，则第二步永远不会执行，就会形成数据库的不一致性。
    • 问题的解决：必须有事务的概念和原子性

    完整性检查和保证

    • 系统自动在删除主表元组时删除子表对应的元组
    • 系统检查子表中是否有相应的元组，如果存在，禁止删除动作

    视图

    特点：

    • 虚表，是从一个或几个基本表（或视图）导出的表
    • 只存放视图的定义，不会出现数据冗余
    • 基表中数据发生变化，从视图中查询的数据也改变

    基于视图的操作：

    • 提供了一种观察数据的逻辑窗口。对视图的操作意味着对基表进行相对应的操作。但对视图是可以有限制的
    • 视图的定义是递归的，可以定义基于该视图的新视图
    CREATE VIEW <视图名> [(<列名> [,<列名>]…)]
    AS <子查询>
    [WITH CHECK OPTION]
    • SELECT语句表示子查询，但通常不允许包括ORDER BY子句和DISTINCT短语
    • WITH CHECK OPTION通过视图进行增删改操作时，不得破坏视图定义中的谓词条件（即子查询中的条件表达式）
    • DBMS执行CREATE VIEW语句时只是把视图的定义存入数据字典，并不执行其中的SELECT语句。在对视图查询时，按视图的定义从基本表中将数据查出（懒查询）
    CREATE VIEW dbo.ShipStatusView AS
    SELECT OrderID, ShippedDate, ContactName FROM Customers c, Orders o
    WHERE c.CustomerID = o.CustomerID and RequiredDate<ShippedDate;

    组成视图的属性列名全部省略或全部指定：

    • 省略：由子查询中SELECT目标列中的诸字段组成
    • 在下列情况下必须命名CREATE VIEW中的列
    • 当列是从算术表达式、函数、常量派生
    • 两个或更多的列可能会具有相同的名称（通常是因为连接）
    • 视图中的某列被赋予了不同于派生来源列的名称

    WITH CHECK OPTION选项：通过视图插入、删除、修改元组时检查元组是否满足视图定义中的条件，如果不满足则拒绝执行。如果视图定义中含有条件，建议选择WITH CHECK OPTION选项，以约束更新数据

    CREATE VIEW Sage_23 AS
    SELECT * FROM Student WHERE Sage < 23
WITH CHECK OPTION

    当通过视图更新学生元组时，系统将检查所更新的学生年龄是否小于23岁，不满足系统将拒绝更新操作

    CREATE VIEW 'D-Sage' (Sdept, Avgage) AS
    SELECT Sdept, AVG(Sage) FROM Student
        GROUP BY Sdept;

    带表达式视图必须明确定义组成视图的各个属性列名。而且由于有集函数，视图无法对数据进行修改

    CREATE VIEW CS_SC(Sno, Sname, Grade) AS
    SELECT Student.Sno, Sname, Grade FROM Student, SC
    WHERE Sdept = '计算机' AND Student.Sno=SC.Sno AND SC.Cno='C2';

    基于视图的视图：

    CREATE VIEW CS_90 AS
    SELECT Sno, Sname, Grade FROM CS_SC  
    WHERE Grade>=90;

    以SELECT *创建的视图可扩展性差，应尽可能避免。因为修改基表结构后，Student表与视图的映象关系被破坏，导致视图不能正常工作

    删除视图

    DROP VIEW CS_90

    删除基表时，必须显式删除基于此基表的视图（或用CASCADE）

    视图查询的方法：

    1. 实体化视图（View Materialization）：有效性检查（检查视图是否存在），执行视图定义，将视图临时实体化，生成临时表，查询视图转换为查询临时表，查询完毕删除被实体化的视图（临时表）
    2. 视图消解法（View Resolution）：进行有效性检查，存在则从字典取出视图的定义，把视图中的子查询与用户的查询结合，转换成等价的对基本表的查询，执行修正后的查询。过程中没有生成表，是直接对原始表的查询

    有些情况下视图消解法不能正确查询，DBMS会限制这些查询。简单的视图消解是总能进行的。但含有聚集函数的视图消解可能会发生错误，只能采取对基表直接操作的方式

    某些视图无法正确更新，例如采用了AVG函数等、删除涉及多个表时。例如：

    CREATE VIEW CS_SC(Sno, Sname, Grade) AS
    SELECT Student.Sno, Sname, Grade 
        FROM Student, SC
        WHERE Sdept='计算机' AND Student.Sno=SC.Sno AND SC.Cno='C2'

DELETE FROM CS_SC
    WHERE Sname='刘明亮'

    视图的优点：

    • 提供了数据的逻辑独立性，使用户外模式保持不变
    • 简化了用户视图，能够简化用户操作，适当的利用视图可简化查询
    • 能让用户以不同角度看待相同的数
    • 提供了安全性

    SQL的数据控制

    授权GRANT，回收语句REVOKE

    GRANT {<权限1>, <权限2>, ...}
    ON TABLE<表名或视图名>
    TO {<用户名1>, <用户名2>, ... | PUBLIC}
    [WITH GRANT OPTION]

    有WITH GRAND OPTION时，被授权的用户可以把获得的权限再授予给其他用户

    一个GRANT语句可以把相应权限同时授予多个用户，用户名表如短语PUBLIC代替，则表示把权限授予所有数据库用户

    GRANT SELECT, DELETE
    ON TABLE Student
    To User2
    WITH GRANT OPTION;

    收回授权

    REVOKE {<权限1>, <权限2>, ...}
    ON TABLE <表名或视图名>
    FROM {<用户名1>, <用户名2>, ... | PUBLIC}
    [CASCADE|RESTRICT]

    例如：

    REVOKE SELECT, DELETE
    ON TABLE Student
    FROM User2
    CASCADE;
