    线性表

    定义与特点

    有长度属性

    定义线性表为n个数据元素的一个有限序列，记为

    \[L = (\alpha_1, \alpha_2, ..., \alpha_n)\]

    其中L为表名，\(\alpha\)为表中元素，是不可再分的原子数据，也称为节点或表项。特点：

    • 存在唯一的第一个元素与最后一个元素
    • 除第一个元素外，其余元素有且仅有一个直接前驱
    • 除最后一个元素外，其余元素有且仅有一个直接后继
    • 每个元素都必须具有相同的数据类型，且不能按子表那样再分割
    • 线性表中每个元素都有位置和值，即下标与具体内容
    • 线性表中元素值可以与位置有关系（例如有序表，按某种规律排序那种），也可以没有关系（例如无序表）

    主要操作

    void initList(List &L);
void clearList(List &L);
bool is_empty(List &L);
bool isFull(List &L);
int Length(List &L);
position Locate(int i, List &L);
position Search(DataType x, List &L);
int Insert(List &L, int i, DataType &x);
int Remove(List &L, int i, DataType &x);
void Traverse(List &L);
void CopyList(List &L, List &L1);
void Sort(List &L);

    顺序表

    定义与特点

    把线性表中的所有元素按照其逻辑顺序依次储存在一块连续的存储空间中，就得到顺序表。特点：

    • 各个元素的逻辑顺序与其存放的物理顺序一致，即第i个元素存储于第i个物理位置
    • 既可以顺序访问（从第一个元素开始逐个访问），也可以随机访问（直接通过下标访问）

    结构定义

    静态存储表示

    • 存储线性表的一维数组大小maxSize和空间data都在结构声明中明确指定
    • 程序编译时数组空间由编译器固定分配，程序退出后此空间自动释放
    • 数组中可以按数组元素的下标直接存取，所花费时间相同
    • 一旦数据空间占满，会导致溢出
    #define maxSize 100
typedef int DataType;
typedef struct {
    DataType data[maxSize];
    int n;
} SeqList;

    其中n为当前已有元素数

    动态存储表示

    • 在程序执行过程中动态分配
    • maxSize放在结构内定义，方便后续更改
    #define initSize 100
typedef int DataType;
typedef struct node{
    DataType *data;
    int maxSize;
    int n;
} SeqList

    数组存储表示

    对应静态存储表示

    • 可以用C的数组实现，可静态分配也可动态
    • 线性表存储数组的数据类型就是顺序表中元素的数据类型，数组的大小要大于等于顺序表的长度
    • 顺序表中第i个元素被存储在数组中下标为i-1的位置上
    • \(Loc(i) = Loc(1) + (i-1)*sizeof(DataType)\)

    顺序表和数组的区别：

    • 顺序表是一种数据结构，而数组是一种编程语言提供的数据类型
    • 顺序表比数组多出一个一对一的逻辑关系，即只能是一维的（要满足前文所说的后继和前驱那个关系）
    • 顺序表长度可以是可变的，而数组长度是固定的（声明时就需要确定）

    性能分析

    查找

    在等概率情形下，平均数据比较次数ACN为：

    \[ACN = \sum p_ic_i = \sum \frac i n = \frac {1+n} 2\]

    平均会查找\(\frac {1+n} 2\)次。在查找不成功时，会将整个表遍历一遍

    插入

    插入一个元素时，会使其后面所有元素向后移动。可以分析平均移动次数AMN

    删除

    删除一个元素时，会使后面所有元素向前移动。可以分析平均移动次数AMN

    单链表

    定义与特点

    为每个数据元素附加了一个链接指针，各个数据元素可以相继存储也可以不相继存储。为每个元素附加了一个指针，形成节点。每个节点都包含数据域与指针域。指向第一个节点的指针称为头指针

    特点：

    • 逻辑顺序可能与物理顺序不一样
    • 长度可扩充
    • 遍历或查找只能从首元结点开始，跟随指针逐个访问，而不能直接访问某节点。即只能顺序存取
    • 当插入或删除时，只需要修改相关节点的指针即可

    常见操作

    List make_empty(List l);
bool is_empty(List l);
bool is_last(Position p, List l);
Position find(ElementType x, List l);
Position find_previous(ElementType x, List l);
void remove(ElementType x, List l);
void insert(ElementType x, List l, Position p);
void delete_list(List l);
Position header(List l);
Position first(List l);
Position advance(Position p); // 找p的前驱节点
ElementType retrieve(Position p);

    其中删除和插入都只需要修改指针即可

    结构定义

    typedef int DataType;
typedef struct node {
    DataType data;
    struct node * link;
} LinkNode, *LinkList;

    其中LinkList是指向首元结点的指针。这是一个递归定义

    带头节点链表

    头结点data域可以不存储任何信息，也可以放特殊标志或表长。好处：

    • 统一了空表与非空表的操作
    • 简化了链表的插入与删除

    尾递归

    例如正向输出单链表：

    void print(LinkList& current) {
    if (current == NULL) {
        printf("\n");
        return;
    }
    printf("%d", current->data);
    print(current->link);
}

    缺点：会有额外的存储开销（递归工作栈等）

    还有其他应用，例如：

    • 尾插法创建链表
    • 前插法创建链表

    可以使用循环将尾递归转换为非递归

    变形

    循环链表

    表尾节点的link指向头结点。只需要知道链表中任意节点的地址，就能访问整个链表。判断是否为尾变为判断p->link == first

    若经常在表头执行操作，可以不用表头指针，只需要一个表尾指针rear来标识，方便进行在最前端最末端插入删除

    应用：求解Josephus问题

    双向链表

    目的是解决在链表中访问直接前驱和直接后驱的问题。在双向链表中每个节点都有两个指针，一个指向直接前驱，一个指向直接后继

    静态链表

    为一维数组中每个元素附加一个链接指针，形成静态链表。允许不改变各元素的物理位置，只需要重新连接就能改变这些元素的逻辑顺序。整个运算过程中存储空间大小不变（因为用数组实现的）

    十字链表

    顺序表与链表比较

    存储方面

    存储密度=数据元素占用存储字节数/为结构分配的最大字节数

    顺序表存储密度为1，因为表示元素间逻辑关系无需用到额外存储空间。单链表的小于1，因为用到了链接指针表示逻辑关系

    空间限制

    顺序表，在静态时无法扩充，动态时可扩充，但过程中新老空间并存，且要移动大量元素。单链表节点只有在使用时才分配

    存取方面

    • 顺序表结构简单，而单链表只能先通过访问某元素之前的元素才能访问之后的元素
    • 顺序表存取灵活，而单链表只能顺序存取
    • 顺序表插入删除消耗较大，而单链表更少

    安全方面

    若表头指针被保护的较好，则单链表更安全
