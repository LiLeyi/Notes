    树

    树

    定义

    自由树：见图论

    有根树：一棵树称为有根树T，定义为n个节点的有限集合，记为

    \[T = \{r, T_1, T_2, ..., T_n\}\]

    其中，r是T的根节点，T_1,T_2,T_3...T_n为互不相交的子集合，每个子集也是一棵树，称为根的子树。每棵子树的根节点有且仅有一个直接前驱（称为父节点，但可以有0或多个直接后继（即下层节点

    基本概念

    节点：

    • 节点：包含数据元素的值及指向其他节点的分支指针
    • 度：某个节点的子树数量（即向下连线数量）
    • 树的度：最大的度

    节点分类：

    • 叶子节点（终端节点）：度为0
    • 分支节点：除叶节点外的其他节点

    节点间关系：

    • 兄弟节点：有相同父节点的节点
    • 父节点：这个节点的上面的节点
    • 子节点：指某节点子树的根节点。这个节点的下面的节点
    • 祖先节点：从根节点到该节点的路径上的所有节点（包括自身）
    • 真祖先节点：从根节点到该节点的路径上的所有节点（但不包括自身）
    • 子孙节点：指某节点以及其子树中所有节点
    • 真子孙节点：指某节点以及其子树中所有节点（但不包括自身）

    路径及相关数量特征：

    • 路径：从a到b的路径，指从a经过一系列节点到b，这一系列节点的集合，唯一
    • 路径长度：路径节点数-1，即路径边的数量
    • 深度：即所处深度，根节点为1，每往下一层加1
    • 高度：空树高度为0，叶节点高度为1，非叶节点的高度等于max(子树高度)+1

    树的数量特征：

    • 树的高度（深度）=根节点高度=最深叶节点深度（等于边的个数，例如若有五层，则深度为4）
    • 宽度：节点数最多那一层的节点数

    树的分类：

    • 有序树：子树之间有顺序关系
    • 无序树：子树之间顺序不重要

    森林：m棵树的集合

    基本操作

    void InitTree(Tree &T);
void ClearTree(Tree &T);
position FirstChild(Tree &T, position p);
position NextSibling(Tree &T, position p);
position Parent(Tree &T, position p);
bool InsertChild(Tree &T, position p, TElemType x); // 在节点p下方插入
bool DeleteChild(Tree &T, position p, int i); // 删除节点p的第i个子节点
void DeleteSubTree(Tree &T, position p);
void Traversal(Tree &T, position p) // 遍历以p为根节点的子树

    树的表示

    双亲表示

    使用一组连续的存储单元存放节点，每个节点两个域：data域、parent域（存放parent节点在数组中的索引）。根节点parent域为-1

    一般存放顺序不作要求，但也可以按照前序、后序等顺序进行存放

    结构定义：

    #define maxSize 50
#define maxStack 50
typedef char TElemType;
typedef struct node {
  TElemType data;
  int parent;
} PTNode;
typedef struct {
  PTNode tnode[maxSize];
  int n;
} PTree;

    找父节点较多时可以使用该结构。但该结构找子节点需要遍历数组，时间复杂度O(n)。但若按照前序排序存入，那么该节点的孩子节点一定紧跟在后面；若按照层序排序存入，则兄弟节点一定紧跟在后面，孩子节点一定在后面的某处聚集

    子女链表表示

    为树中每个节点设置一个子女链表

    #define maxNodes 30
typedef char TElemType;
typedef struct enode { // 子女链表节点
  int child;
  struct enode *link;
} ENode;
typedef struct vnode { // 树节点
  TElemType data;
  int parent;
  struct enode *first;
} VNode;
typedef struct { // 子女链表存储表示（不太懂这个啥意思）
  VNode NodeList[maxNodes];
  int n;
} ChildList;

    左孩子右兄弟链表表示

    也称为树的二叉树表示，每个节点度为2，是最省存储空间的存储方式

    每个节点由三个域构成：lchild,data,rsibling。其中lchild存指向左边第一个孩子的指针，data存数据，rsibling存指向右边的兄弟的指针

    这种表示方法可以将一个树转化为二叉树

    对于森林转二叉树，将森林的所有根节点看做兄弟，用该表示法即可

    逆向即可将二叉树转为树或森林

    结论

    • 有n个节点的树有n-1条边

    遍历

    前序遍历

    是一种深度优先遍历

    void PreOrder(BiTNode *T) {
  if (T != NULL) {
    printf("%c", T->data);
    PreOrder(T->lchild);
    PreOrder(T->rchild);
  }
}

    可用于：

    • 建立树
    • 输出树的凹入表示
    • 求深度
    • 判断两棵树是否相似、相同

    后序遍历

    是一种深度优先遍历

    void PostOrder(BiTNode *T) {
  if (T != NULL) {
    PreOrder(T->lchild);
    PreOrder(T->rchild);
    printf("%c", T->data);
  }
}

    可用于：

    • 求高度
    • 销毁树

    层序遍历

    是一种广度优先遍历

    void LevelOrder(BiTNode *T) {
  if (T == NULL) return;

  queue<BiTNode*> q;
  q.push(T);

  while (!q.empty()) {
    BiTNode *current = q.front();
    q.pop();

    printf("%c", current->data);

    if (current->lchild != NULL)
      q.push(current->lchild);
    if (current->rchild != NULL)
      q.push(current->rchild);
  }
}

    可用于：

    • 建立树
    • 求每层节点数，求树宽度
    • 判断树是否为完全二叉树

    二叉树

    定义与特点

    一棵二叉树是节点的一个有限集合，该集合或为空，或是由一个根节点加上两棵分别称为左子树和右子树的互不相交的二叉树组成：

    \[T = \begin{cases} \phi, &n = 0, \\ \{r, T_L, T_R\}, & n>0 \end{cases}\]

    节点的度最大为2。特点：

    • 为有序树，区分左右子树
    • 定义是递归的（和树一样）
    • 二叉树只有五种形态：空、只有根节点、只有左子树、只有右子树、左右子树都存在
    • 二叉树不是树，因为树是一种图，图的节点非空，但二叉树有为空的情况。非空二叉树有左右子树的区别，即使一边为空，另外一边也要保持左右之分。二叉树属于N叉树
    • 树的术语都可以用在二叉树上

    会有n+1个空指针，n-1个边

    性质

    • 二叉树第i层最多有\(2^{i-1}\)个节点\(i\geq 1\)
    • 深度为k的二叉树最少有k个节点，最多有\(2^{k}-1\)
    • 对非空二叉树，若叶子结点数为\(n_0\)，度为2的非叶节点数为\(n_2\)，则\(n_0 = n_2 + 1\)

    设二叉树高度为h

    • 完美二叉树（满二叉树）即所有节点要么是叶节点，要么度数为2
    • 完全二叉树：除了最底层外，其他层均为满的，且最底层节点都靠左排列。节点个数在区间\([2^h,2^{h+1}-1]\)。即高度可以由节点N进行计算，\(h=[logN]+1(n\neq 0)\)或\(log_2(n+1)\)向上取整。可按层序遍历，将其存储为数组。某个节点的索引乘2，可以得到其左子树的索引。可以通过除二向下取值得到父节点

    对于完全二叉树，可由索引通过计算得其是左孩子还是右孩子、所处层数、左孩子与右孩子索引、左右兄弟索引等

    主要操作

    void InitTree(Tree &T);
void ClearBTree(Tree &T);
BiTNode *LeftChild(BiTNode *p);
BiTNode *RightChild(BiTNode *p);
BiTNode *Parent(BiTNode *p);
bool GetData(BiTNode *p, TElemType &x);
void CreateBiTree(BiTNode &T);
void PrintBinTree(BiTNode &T);
int Height(BiTNode *T);
void PreOrder(BiTNode *T);
void InOrder(BiTNode *T);
void PostOrder(BiTNode *T);
void LevelOrder(BiTNode *T);

    存储表示

    顺序存储表示

    对于完全二叉树，对节点编号，存入数组即可

    对于一般二叉树，按完全二叉树的编号进行储存（空的编号空出来即可）。可能导致很大的空间浪费

    结构定义

    #define maxSize 100
typedef char DataType;
typedef struct {
  DataType data[maxSize];
  int n;
} SqBTree;

    该存储方式对于完全二叉树最好，对于一般二叉树不适用

    链式存储表示

    每个节点包含三个域：数据域data，左孩子指针域lchild，右孩子指针域rchild。这种链表称为二叉链表，可以很方便找到子节点，但找父节点较为困难（可以再添加一个域存放parent指针，称为三叉链表）

    整个二叉树的链表有一个树根指针，指向二叉树根节点

    在含n个节点的二叉树中，有n+1个空指针域，因为2n个指针域中只有n-1个指针域存有边信息。三叉链表则有n+2个空指针域

    二叉或三叉链表也可以是静态链表结构

    二叉链表结构定义：

    typedef int DataType;
typedef struct node {
  DataType data;
  struct node *lchild, *rchild;
} BiTNode, *BinTree;

    三叉链表比较耗空间，所以只需要在程序中添加一个栈，记录路径，就可以在查找时顺便输出父节点

    遍历

    其余遍历与树相同，但多了中序遍历（对称序遍历）

    void InOrder(BiTNode *T) {
  if (T != NULL) {
    InOrder(T->lchild);
    printf("%c", T->data);
    InOrder(T->rchild);
  }
}

    结论：若某叶节点是某二叉子树中序序列最后一个节点，则一定是前序遍历最后一个节点。逆命题不成立

    逆命题的反例：

    A
 \
  B
 / \
C   D
   /
  E

// 对于以A为根的整棵树：
// - 中序遍历（左-根-右）：A-C-B-E-D （最后一个节点是D）
// - 前序遍历（根-左-右）：A-B-C-D-E （最后一个节点是E）

    前序、中序、后序遍历都可以统一为欧拉巡回算法

    void EulerTour(BiTNode &T) {
  visit(T);
  if (T->lchild != NULL) EulerTour(T->lchild);
  visit(T);
  if (T->rchild != NULL) EulerTour(T->rchild);
  visit(T);
}

    可以利用栈将前序、中序、后序遍历改为非递归算法。

    唯一确定二叉树：

    • 中序+后序
    • 中序+先序
    • 中序+各节点父节点指针
    • 中序+层序
    • 中序+各节点层次

    计数

    在前序序列一定时，中序序列有\(\frac 1 {n+1}C_{2n}^n\)种（卡特兰数）

    可通过遍历时用到栈，转化为退栈序列问题

    线索二叉树

    定义

    对于遍历产生的序列，除了序列第一与最后一个元素，其余元素都有前驱和后继（在不同的情况下称为前序前驱、前序后继、中序前驱、中序后继、后序前驱、后序后继、层次前驱、层次后继）

    若希望能很快找到某节点的前驱和后继，而不用每次都遍历这个二叉树，则可以：

    • 每个节点增加前驱指针域、后续指针域。但浪费空间，一般不用这种
    • 空指针与存放信息，空lchild存放前驱节点指针、空rchild存放后续节点指针（这两种指针称为线索）

    加上线索的二叉树称为线索二叉树。按照不同的遍历序列可分为四种不同线索二叉树：中序二叉树、前序二叉树、后序二叉树、层次二叉树。节点构造：

    typedef char DataType;
typedef struct TRNode {
  bool ltag, rtag; // 标志域，false表示为孩子，true表示为线索
  struct TRNode *lchild, *rchild;
  DataType data;
} ThreadNode;

    中序线索二叉树

    二叉树的中序线索化：对遍历函数加入一个前驱指针即可

    中序二叉树下，中序、前序、后序遍历不用栈也能通过循环实现

    一些遍历操作：

    ThreadNode * inFirst(ThreadNode * t) { //求*t为根的线索树上中序第一个节点
  while (t->ltag == false) t = t->lchild; // 最左下节点（不一定为叶节点）
  return t;
}

ThreadNode * inLast(ThreadNode * t) { //求*t为根的线索树上中序最后一个节点
  while (t->rtag == false) t = t->rchild; // 最右下节点（不一定为叶节点）
  return t;
}

ThreadNode * inNext(ThreadNode * t) { // 求*t的中序后继
  if (t->rtag == false) return inFirst(t->rchild);
  return t->rchild; // 这种情况是右边是线索，直接输出
}

ThreadNode * inPrior(ThreadNode * t) { // 求*t的中序前驱
  if (t->ltag == false) return inLast(t->lchild);
  return t->lchild; // 这种情况是左边是线索，直接输出
}

void Inorder(ThreadNode *t) { // 中序遍历
  for (ThreadNode *p = inFirst(t); p != NULL; p = inNext(p))
    printf("%c", p->data);
}

void InorderReverse(ThreadNode *t) { // 逆中序遍历
  for (ThreadNode *p = inLast(t); p != NULL; p = inPrior(p))
    printf("%c", p->data);
}

    广义表

    定义

    广义表不仅可以表示树（因为有递归表）

    一个广义表可定义为n个表元素\(a_0,\dots,a_{n-1}\)组成的有限序列，记作

    \[LS = (a_0, \dots, a_{n-1})\]

    其中LS为表名，\(a_i\)是表中元素，或者是数据元素（称为原子元素）或者是子表。n是表的长度，即表中元素个数。空表长度为0。表的长度一定有限

    \((a_1,a_2,\dots,a_n)\)，\(a_1\)为表头，其余形成的表为表尾。（注意层次不一样。例如(a,b)，取表头为a，取表尾为(b)）

    • 括号层数为深度（原子表为0，空表为1），最外层包含元素个数为长度。
    • 广义表可以是自身的子表，有这种性质的表称为递归表
    • 可共享：例如E(B(a,b),D(B(a,b),C(c)))，此时B称为共享表或再入表
    • 广义表有层次，有序，并且是有向的
    • 广义表只含原子时，退化为线性表

    表示

    头尾表示

    原子节点的构造：tag=0,data

    表节点的构造：tag=1,hlink,tlink

    实用性不强

    拓展线性链表表示

    节点构造均为：tag,info,tlink。对于原子节点，tag=0，info存储数据，tlink存储指向同层下个表元素节点的指针；对于表节点，tag=1，info存储表头节点指针

    这种表示存在缺点，每个对子表的指针没有指向“头节点”，导致首元结点删除与插入困难

    实用性不强

    层次表示

    有三种节点

    • 原子节点：存放原子的值（value）与同层下一个元素节点的指针（tlink）
    • 子表节点：存放指向子表头节点的指针（hlink）和同层下一元素节点的指针（tlink）
    • 头节点：标志广义表，存放表名（name）和指向该广义表表头元素节点的指针（tlink）

    每个节点也有一个tag，标志节点种类

    最优树

    概念与定义

    概念：

    • 节点的路径长度：在一条路径中，每经过一个结点，路径长度都要加1
    • 结点的权：给每一个结点赋予一个新的数值，被称为这个结点的权
    • 结点的带权路径长度：指的是从根结点到该结点之间的路径长度与该结点的权的乘积
    • 扩充n叉树：给叶节点全部赋权的n叉树
    • 树的带权路径长度为树中所有叶子结点的带权路径长度之和。通常记作 “WPL” 。使用后序遍历计算即可

    在所有含n个叶子结点，并带相同权重的m叉树中，必然存在权值最小的m叉树（因为情况有限）

    哈夫曼树与生成算法

    哈夫曼树指最优扩充二叉树。构造：哈夫曼算法

    • F中选两个权重最小的节点合成二叉树，根节点权重为两权重之和
    • 新树加入F，删除两个旧节点
    • 再在F中找两个最小值合并，删除两个旧元素
    • ...

    哈夫曼编码为前缀编码（进行解码时不产生二义性，确保正确解码。即任一一个编码不为另外一个编码的前缀）

    结构定义：采用静态三叉链表，每个节点多出一个weight域

    #define leafNumber 20
#define totalNumber 39
typedef struct {
  char data;
  int weight;
  int parent, lchild, rchild;
} HTNode;
typedef struct {
  HTNode elem[totalNumber];
  int num, root;
} HFTree

    最优判定树

    哈夫曼算法可以保证WPL最小，但不能保证比较次数最小，也不能保证输入的排序（例如输入ABC，在叶节点中也保持从左到右是ABC）

    使用Hu-Tucker算法：类似哈夫曼算法，但找最小值合并过程限制在相邻两个树上（例如A，B，C，D四棵树，只能对比A+B与B+C与C+D的权重和，而不是哈夫曼算法中随意取最小权重和）

    堆

    前置知识

    设二叉树高度为h

    • 完美二叉树即所有节点要么是叶节点，要么度数为2
    • 完全二叉树：除了最底层外，其他层均为满的，且最底层节点都靠左排列。节点个数在区间\([2^{h-1},2^{h}-1]\)。即高度可以由节点N进行计算，\(h=[logN+1]\)。可按层序遍历，将其存储为数组。某个节点的索引乘2，可以得到其左子树的索引。可以通过除二向下取值得到父节点
    • 最小树：任意父节点小于子树中任意值

    定义

    最小堆同时为完全二叉树和最小树。同样可以定义最大堆

    小根堆的结构定义：

    #define heapSize 40
typedef int HElemType;
typedef struct {
  HElemType *elem;
  int n, maxSize;
} minHeap;

    存储数组elem是完全二叉树的储存顺序。若第一个索引为0，则：

    • 若i=0，则为根节点
    • 若i>0，i的父节点索引为\([(i-1)/2]\)
    • 若2i+1>n-1，则节点i无左孩子；否则i的左孩子索引为2i+1
    • 若2i+2>n-1，则节点i无右孩子；否则i的右孩子索引为2i+2

    常见操作

    以最小堆为例

    • 插入时，将元素放在堆的末尾，然后向上调整。与父节点判断大小，若需要交换则交换，接下来继续判断该元素与现在的父节点...（上滤）。时间复杂度O(logN)
    • 删除最小节点时，删除根节点。将最大的节点移动到根节点，与子树进行对比移动（下滤）
    • 增减优先级：直接增减，然后进行下滤或上滤
    • 删除某节点：将该节点变成一个很小的值，然后删除最小值节点
    • 创建：先直接创建一个完全二叉树，从\([N]\)的位置进行下滤，然后对\([N-1]\)进行下滤，一直对0进行下滤。时间复杂度\(O(N)\)。要进行\(N/2\)次筛选
