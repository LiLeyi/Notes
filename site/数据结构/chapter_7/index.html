    排序

    通用接口：

    void X_Sort(ElementType A[], int N);

    冒泡排序

    不涉及额外空间，时间复杂度O(n^2)

    选择排序

    i=0，从数组后面选择最小（最大）元素，插到第i个位置，然后i++，直到遍历整个数组

    插入排序

    将数组分为两部分，一部分为已排序，另一部分未排序，考虑交界处的元素，依次向已排序部分对比、交换，直到找到插入位置

    在较为有序时，时间复杂度接近O(n)

    描述

    逆序对

    随机数组中逆序对个数的期望是\(\frac {N(N-1)} 4\)。证明：任意序列与其倒序的序列，逆序对个数之和是\(C_N^2\)，所以再除二得\(\frac {N(N-1)} 4\)。所以任意通过交换相邻元素的排序算法，在平均情况下，时间复杂度至少需要\(\Omega(N^2)\)

    Shell排序

    进行\(n_t-sort\)，每隔n-1个元素，形成子数组，对子数组进行排序（可以用其他方法排序）。再进行\(n_{t-1}-sort\)...最后进行1-sort。要求\(n_t > n_{t-1} > n_{t-2} > ... > 1\)，最后一个一定要为1

    子序列排序最好使用插入排序，因为在较为有序时其时间复杂度接近O(n)

    • 希尔增量序列：\(n_t = [N/2], n_k = [n_{k+1} / 2]\)
    • Hibbard增量序列：\(n_k = 2^k - 1\)。为了改进希尔增量序列的，例如4-sort过了后为2-sort的问题。Hibbard中\(n_i\)互素。最坏情况下，时间复杂度为\(\Theta(N^{3/2})\)，最好时\(O(N^{5/4})\)
    • Sedgewick序列\(9\times 4^i - 9 \times 2^i + 1\)，\(O(N^{7/6})\)，即\(\{1,5,19,41,109,\dots\}\)，最差\(O(N^{4/3})\)

    对数据量较大时比较好用

    堆排序

    算法1

    建立一个堆，以及一个临时数组，每次从堆中取最小值，放到临时数组中。最后临时数组为排序后数组

    • 建堆O(logN)
    • 删除最小值O(logN)，循环N次，则为O(NlogN)
    • 拷贝O(1)

    时间复杂度为O(NlogN)，但有额外的空间复杂的O(N)

    算法2

    建立大顶堆，逐步删除最大值。删除元素的过程，删除空出来的空间放刚删除的元素

    此时无额外时间复杂度

    归并排序

    合并两个数组：使用两个游标，在两个数组的最左边，进行比较、移动

    合并操作时间复杂度\(O(NlogN)\)，但有更多的空间复杂度

    快速排序

    步骤

    用两个游标，一个从左到右，一个从右到左，如果左侧游标元素大于右边，则对调。令i为左侧游标，j为右侧游标

    1. 若i的数据比枢轴小，跳过
    2. 若j的数据比枢轴大，跳过
    3. 如果\(i<j\)，则对调。否则break
    4. 返回步骤1进行迭代
    5. 若break，则交换pivot和j

    枢轴选取：

    • 从两端开始，但在有序时，时间复杂度为\(O(N^2)\)
    • 随机选择，但随机数发生器耗时较大
    • 三数中值分割法：选择左右两个元素，计算中间位置，将这三个元素先变为有序，将center放到right-1位置（因为right已经比center大了），返回center作为枢轴

    优化：

    • 当数组过小时，调用其他排序

    分析

    \[T = T(i) + T(N-i-1) + c N\]

    最坏情况

    \[T(N) = T(N-1) + cN\]

    达到\(O(N^2)\)

    最好情况

    \[T(N) = 2T(N/2) + cN\]

    为\(O(NlogN)\)

    平均情况：

    \[ET(i) = E(N-i-1) = \frac 1 N (\sum_{i=0}^{N-1} T(j))\]

    带入可得\(T(N) = O(NlogN)\)

    表排序

    不移动元素本身，而只移动元素本身的排序方法，我们称之为“间接排序”。

    根据指定key排序指向元素的指针

    桶排序

    将数组分为多个桶，每个桶内进行排序，最后将桶进行排序合并

    MSD：先按照高优先级分类，建桶，再在6桶内排序。

    基数排序：或者先按低优先级建桶，然后按顺序存入，再按照次低优先级建桶，按低优先级桶中顺序存入...直到最高优先级
