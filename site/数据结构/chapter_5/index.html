    查找

    顺序查找

    在建立顺序表、线性链表时，可以通过将访问频率高的元素放在前面以达到降低平均查找次数

    顺序表中，可在末尾放一个“监视哨”，检测到时返回该监视哨，可以在每次迭代中减少一次对比时间.也可以使用递归进行查找

    对于线性链表，也可以迭代、递归遍历查找

    折半查找

    也称为二分查找、对分查找。在有序顺序表中（我们这里以从小到大为例），先求出查找区间正中元素下标mid，利用其键与目标比较：

    • 若相等，查找成功
    • 若\(x<data[mid]\)，则将区间缩小到前半区间
    • 若\(x>data[mid]\)，则将区间缩小到后半区间

    每比较一次，区间缩小一半。时间复杂度\(O(logn)\)，若使用递归，则空间复杂度也为\(O(logn)\)

    性能分析：

    • 最多查找\(log_2(n+1)\)向上取整次
    • 查找成功平均长度\(\frac {n+1} n log_2(n+1) - 1 \approx log_2(n+1) - 1\)
    • 查找失败平均长度...

    对于含重复元素的有序序列...

    斐波那契查找

    看书，按斐波那契数列进行划分，类似折半查找

    n较大时，称为黄金分割法，平均性能好于折半查找。但最坏情况下，性能比折半查找差。查找成功的平均次数也是\(O(log_2 n)\)

    插值查找

    对给定的有序数组进行插值，得到\(K(x)\)，将要查询的值y输入得到其大致位置\(y=K(x_0)\)，其中\(x_0\)为所求。

    进行线性插值时，区间中“中点”计算公式为

    \[mid = low + \frac {K[mid] - K[low]} {K[high] - K[low]} (high - low)\\ \approx low + \frac {y - K[low]} {K[high] - K[low]} (high - low)\]

    对较小表，折半查找更好。但对于大表、开销很大、访问成本高的表，插值查找更好

    二叉查找树

    定义

    也称为二叉排序树。是空树，或是具有以下性质的二叉树

    • 非空左子树的所有键值小于其根节点的键值（关键码）
    • 非空右子树的所有键值大于其根节点的键值（关键码）
    • 左右子树都是二叉搜索树

    作为查找的依据，二叉查找树上每个节点都有能够用于识别它的数据项，称为关键码或关键码项，在应用程序中实际被赋予的值称为关键码值，简称关键码。利用它可以进行对一组数据进行排序，因为只需要对一棵二叉查找树进行中序遍历，即可依据关键码的值从小到大将各节点排序起来

    结构定义

    typedef int TElemType;
typedef struct tnode {
  TElemType data;
  struct node *lchild;
  struct node *rchild;
} BSTNode, *BSTree;

    常见操作

    • 查找（递归非递归都能实现）
    • 插入：必须先检查在不在树中。插入操作类似查找操作
    • 建立
    • 删除：
    • 左右子树全空：直接删除
    • 一棵子树为空：将不空的子树直接代替该节点
    • 两子树均不空：两种方法
      1. 将其中序前驱代替该节点，然后删除中序前驱（递归）
      2. 将其中序后继代替该节点，然后删除中序后继（递归）

    性能分析

    因为关键码插入顺序不同，共有\(\frac 1{n+1}C_{2n}^n\)个不同的排序二叉树。将已有节点称为内节点，假设每个内节点的空孩子指针有一个虚拟的外节点，将这种加入外节点的二叉查找树称为扩充二叉树（或判定树、决策树）

    一个有n个内节点的二叉树，一定有n+1个外节点

    根据比较次数\(l[i]\)与访问概率\(p[i]\)，可以得出平均查找次数（分成功与不成功）

    随机情况下，二叉树的插入、删除、查询时间复杂度为\(O(logn)\)

    中序二叉查找树

    定义

    兼容了二叉查找树与线索二叉树的特点。结构定义与线索二叉树一致

    常见操作

    • 构建：先按二叉查找树的规则构建，再中序线索化
    • 插入：插入类似排序二叉树，插入后需修改线索
    • 删除：先找到删除节点的中序前驱与后继，修改线索，再删除目标节点
    • 范围查找：利用中序遍历非递归算法遍历即可

    平衡二叉树（AVL Tree）

    定义

    AVL是高度平衡的二叉树，由俄罗斯数学家G.M.Adel'son-Vel'skii与E.M.landis提出。AVL树是为了解决二叉树在极度不平衡时导致的插入、查询、删除耗时过大的问题

    平衡因子\(BF(node) = h_L - h_R\)，AVL树中只能为-1，0，1。空树是高度平衡的（定义空树高度为-1）。如果T是非空二叉树，\(T_L,T_R\)为左、右子树，若满足：

    1. \(T_L,T_R\)都是平衡二叉树
    2. \(h_L - h_R\leq 1\)

    则称T为平衡二叉树。

    对于一个树，若根节点BF<0，则左树比右树低，若BF>0，则左树比右树低

    注意：

    • AVL树不一定是高度最小的二叉查找树
    • AVL树不一定是理想平衡树（理想平衡树的每个节点BF都是0）
    • 插入一个节点后，失衡节点可能多达\(O(log_2 n)\)个

    平衡化旋转

    插入节点时，有可能会导致树不平衡，此时需要从插入节点沿通向根位置的路径回溯，检查BF是否合规，若不合规，则停止回溯，开始进行平衡化旋转。

    插入某个节点不平衡时（称为trouble maker），最近的不符合BF条件的节点称为trouble finder

    若maker在finder的右子树的右子树中，则进行RR rotation，将trouble finder变为原trouble finder的右侧子节点，然后trouble finder变为子节点

    ...左子树的左子树...LL Rotation

    • LL型：右旋失衡节点。失衡节点平衡因子=2，失衡节点左孩子平衡因子=1
    • RR型：左旋失衡节点。失衡节点平衡因子=-2，失衡节点右孩子平衡因子=-1
    • LR型：先左旋失衡节点左孩子，自己再右旋。失衡节点平衡因子=2，失衡节点左孩子平衡因子=-1
    • RL型：先右旋失衡节点的右孩子，自己再左旋。失衡节点平衡因子=-2，失衡节点右孩子平衡因子=1

    删除节点时，需要对每个父节点进行检查，可能不止出现一次失衡。但只需要对最小不平衡子树进行平衡化旋转即可。

    插入时，检查插入节点的父节点平衡因子：

    • 若为0，则不需要回溯
    • 若为\(\pm 1\)，则需要继续回溯，继续考察该节点的父节点
    • 若为\(\pm 2\)，则需要旋转

    删除时，检查删除节点父节点平衡因子：

    • 若原来为0，变为了\(\pm 1\)，则不需要回溯
    • 若原来为\(\pm 1\)，变为0，则需要继续回溯，继续考察该节点的父节点
    • 发生了不平衡，进行平衡化旋转

    性能分析

    已知高度求最小节点数：设\(N_h\)为高度为h的AVL树的最少节点数，在最差情况下，根的一颗子树高度为h-1，另外一颗为h-2，因此在最坏情况下\(N_0 = 0, N_1 = 1, N_h = N_{h-1} + N_{h-2} + 1\)，则有\(N_h = F_{h+2}-1\)成立，其中\(F_n\)为第n项斐波那契数（\(F_0 = F_1 = 1\)）

    已知节点数N：最大高度为\(h_{max} = \frac 3 2 log_2 (N+1)\)，最小高度可套用完全二叉树的公式，\(h_{min} = [log_2 N] + 1\)或\(log_2 (N+1)\)向上取整

    距离根最近的叶节点位置为\(level_h = (h+1)/2\)向上取整，或\(\begin{cases}level_h = h, & h =0,1\\ level_h = min\{level_{h-1}, level_{h-2}\} + 1, & h > 1\end{cases}\)

    红黑树

    条件：

    • 为搜索二叉树
    • 根与叶子（NULL）节点都是黑色
    • 不存在连续两个红色节点
    • 任一节点到叶所有路径黑节点数量一致

    最长路径不会超过最短路径的两倍，即任一节点左右子树的高度相差不超过两倍。所以红黑树查询效率略低于AVL，但插入删除的旋转调整次数小于AVL

    插入：默认为红色，按BST的插入规则进行插入。破坏规则时：

    • 为根节点：直接变黑
    • 叔节点为红色：让父节点、叔节点、祖父节点变色，当前判定节点变为祖父节点，让祖父节点继续判定
    • 叔节点为黑色（NULL也为黑色）：LL、RR后变父节点与祖父节点的颜色，LR、RL后变自己与祖父节点的颜色

    删除：

    • 删除带左右子树的节点，用直接前后驱代替，问题转化为后两种情况
    • 只有左或右孩子：代替后变黑
    • 没有孩子：红节点直接删除，黑节点看视频吧

    Splay Tree

    搜索树，查找时将节点移动到根节点，从而提高查找效率

    操作：

    • 单旋：动目标节点
    • 双旋：有时可以缩短深度
    • 之字形（异构调整。目标在祖父的右子树的左子树，或左子树的右子树）：与AVL树一样，先将父节点旋转，再旋转原来的祖父节点）
    • 一字形（同构调整。目标在祖父的左子树的左子树，或右子树的右子树）：先旋转祖父节点，再旋转父节点

    将查询节点向上若干次双旋，若最后只有一个父节点，再进行一次单旋即可

    对平衡二叉树，提根的时间复杂度为O(logn)

    splay(x,p)，将x旋转到p位置，成为p的子树

    split(p)，按p分裂为左右子树，先把p旋转到根节点，分裂，x属于左子树一般

    merge：合并左右子树，先将右子树最小元素旋转到右子树的根节点，然后把左子树接上去

    insert：按搜索树规则插入到根节点，然后旋转到根节点

    delete：找到要删除的节点，将其旋转到根节点，然后删除根节点，合并左右子树

    B树

    索引顺序表

    当记录个数较大时，可采用索引结构实现储存与查找。索引结构由索引表与数据表组成。数据表存放文件中所有的数据记录，索引表由索引项组成，每个索引项对应一个数据。

    若记录在数据表中按照加入顺序存储时，必须采用稠密索引，这时索引文件称为索引非顺序结构，相应的在索引文件上的查找称为索引非顺序查找

    但当记录在表中按关键码有序存放时，可以把所有n个记录分为b个子表（块）存放，且分块有序（即后一个子表中任意关键码大于前一子表中任意关键码），再建立一个索引表，记录每个子表最大关键码以及其子表在文件中起始位置，这样的索引文件称为索引顺序结构，相应的在索引文件上的查找称为索引顺序查找

    分块查找

    先在索引表中确定子表，再在子表中确定索引位置

    多级索引结构及m叉查找树

    当数据记录特别大时，索引表本身也很大，所以可建立多级索引。可让最高级索引在内存中常驻

    这种多级索引构成一种m叉树，分支节点表示一个索引块，最多存放m个索引项。叶节点存放记录的关键码与存放地址

    多叉平衡搜索树

    访问节点在硬盘进行，节点内操作在内存执行，可加快速度

    • 内部节点：包含数据的节点
    • 外部节点（失败节点）：不包含数据的节点，即最外侧空节点

    满足性质：

    • 平衡：所有内部节点的叶子结点必须在同一层
    • 有序：任何一节点左子树都小于他，右子树都大于他
    • 多路：对于m阶B树，最多m个分支，每一个节点最多m-1个元素。对于根节点，最少2个分支，一个元素；对于其他节点，最少\(m/2\)向上取整个分支，\(m/2\)向上取整-1个元素

    查找...

    插入：先找到位置，检查是否上溢（即是否超过最多元素树），未上溢就插入；上溢了就进行分裂，将中间元素（第\([m/2]+1\)，从1开始算起）添加到父节点，左右分为两个树添加到父节点。可能不止上溢一次，若根节点上溢，就创建新的根节点

    删除：

    • 若在非叶节点中，且有多个元素，直接删除。若只有一个元素，则按BST的规则删除，转化为删叶节点操作（因为要用直接前驱或后驱取替代，递归到最后就要删除叶节点）
    • 若在叶节点，若未下溢出，直接删；若下溢出，先找兄弟借元素（父节点先给当前节点，兄弟节点再给父节点，注意保持有序性）；若不够借，就先把自己给父节点，然后父节点相应元素给兄弟节点进行合并

    下溢可能不止一次，若借完后根节点空了，则层数-1
