    计算复杂性

    时间复杂度

    判定器M的运行时间或时间复杂度是\(f: N\mapsto N\)，\(f(n)\)是在所有长为n的输入上运行的最大步数。若f(n)是M的运行时间，则称M在时间f(n)内运行，或M是f(n)时间图灵机

    例如判定\(\{w | w = 0^{2k}, k \in N \}\)，\(f(2^k) = (2k+1)2^k + 1, f(2n+1) = 2n+2\)。则\(n+1 \leq f(n) \leq 3n log n\)

    O、o记法：对于函数\(f, g: N \mapsto R^+\)

    • \(\exists c, \limsup_{n\to \infty} \frac {f(n)} {g(n)} \leq c\)，则记为\(f(n) = O(g(n))\)
    • \(\lim_{n\to \infty} \frac {f(n)} {g(n)} = 0\)，则记为\(f(n) = o(g(n))\)

    例如\(A = \{0^k1^k| k>0\}\)时间复杂度为\(O(n^2)\)等

    时间复杂性类

    对于函数\(t: N\mapsto N\)，时间复杂性类\(TIME(t(n))\)定义为：

    \[TIME(t(n)) = \{L|\text{存在O(t(n))时间TM判断L}\}\]

    例如\(A = \{0^k1^k| k>0\}\in TIME(n^2)\)，但可以经过优化，得到O(nlogn)的复杂的

    定理：时间\(o(nlogn)\)的单带图灵机判定的语言是正则语言。即

    \[TIME(o(nlogn)) \subseteq \text{正则语言类} \subseteq TIME(n) \subseteq TIME(o(nlogn))\]

    可推出正则语言类=\(TIME(n)=TIME(o(nlogn))\)

    单带与多带图灵机运行时间比较

    设函数\(t(n) \geq n\)，则每个t(n)时间多带TM和某个\(O(t^2(n))\)时间单带TM等价

    非确定型图灵机运行时间

    对非确定型图灵机，运行时间是在所有长为n的输入上，所有分支的最大步数

    设\(t(n)\geq n\)，则每个t(n)时间NTM都有一个\(2^{O(t(n))}\)时间单带确定TM与之等价。\(NTIME(t(n)) \subset TIME(2^{O(t(n))})\)

    P, NP

    素性测试\(Prime = \{p|p\ is\ Prime\}\)

    2001年被证明存在多项式算法

    P是单带确定TM在多项式时间内可判定的问题，即\(P = \bigcup _k TIME(n^k)\)。重要性：

    • 对所有与单带确定TM等价的模型，P不变
    • P大致对应于在计算机上实际可解的问题

    NP是单带非确定图灵机TM在多项式时间内可判定的问题，即\(NP = \bigcup_k NTIME(n^k)\)

    则\(P\subseteq NP \subseteq EXP\)。且\(P\subsetneq EXP\)。目前P是否等于NP未知

    任意非确定型多项式时间问题，都可以被确定图灵机用暴力枚举方式在指数时间内解决

    属于P的问题：

    • 求最大公因子：欧几里得算法、辗转相除法
    • 模p指数运算
    • 上下文无关语言有\(O(n^3)\)判定器
    • 素性测试

    快速验证：给出答案，是否能在多项式时间内验证其是否正确。直观来说，NP是能够快速验证的问题。属于NP的问题：

    • HP哈密顿路径问题
    • CLIQUE，判定一个图中有多少团（完全子图）

    NP完全

    cnf：合取范式k-cnf，每个子句最多包含k个变量的cnf。可满足性问题\(SAT = \{<\phi> | \phi \text{是可满足的布尔公式}\}\)是NP完全的

    二元是P，三元是NP-完全（暂时未知是否属于P）

    二元问题：

    1. 当2cnf中有子句是单文字时，反复执行（直接）清洗：删去含x的子句，删去含\(\neg x\)的文字。若清洗过程反复出现相反单文字子句，则清洗失败并结束
    2. 若无单文字子句，则任选变量，进行真假赋值各一次，进行共两次清洗，若两次清洗都失败，则不可满足
    3. 若成功清洗后有子句剩下，则继续2，否则回答可满足

    规约

    多项式时间可计算函数\(f:\Sigma^* \mapsto \Sigma^*\)，称A可多项式时间映射规约到B（\(A\leq_P B\)），若存在多项式时间可计算函数f，\(\forall w \in \Sigma^*, w\in A \Leftrightarrow f(w)\in B\)，函数f称为A到B的多项式时间归纳。f将A的实例编码转换为B的实例编码（映射规约记为\(\leq_M\)）

    Cook-Levin引理：\(\forall A\in NP, A\leq_P SAT\)

    若证明了\(SAT\in P\)，则NP=P（目前还无证明）

    规约引理：若\(A\leq_P B, B\in P \Rightarrow A\in P\)

    若f是\(n^a\)时间规约，M是\(n^b\)时间判定器，则N的时间\(\leq n^{ab}\)

    定理：

    \[3SAT\leq_P CLIQUE\]

    设有k个子句，则构造无向图，有k组节点，每组3个节点。同组节点无边相连，相反标记（只有\(x_i\)和\(\bar x_i\)不连）无边相连（指有没有非）

    可满足等价于存在k团（每组挑一个真顶点得到k团，非同组非相反）

    且规约时间为多项式

    NP完全性

    语言B称为NP完全，满足：

    • \(B\in NP\)
    • \(\forall A\in NP, A\leq_P B\)
    flowchart TD
A(SAT) --> B(3SAT) --> C(哈密顿路径HP) --> D(UHP无向图哈密顿路径) --> E(UHC无向图哈密顿圈) --> F(TSP旅行商问题)

B --> G(KS。01背包)
B --> H(CLIQUE)

    定理：

    • \(A\leq_P B, B\in P \Rightarrow A\in P\)
    • \(B\)是NPC，\(B\in P\)，则可以推出P=NP（现在无证明）
    • 若B是NPC，\(B\leq_P C, C\in NP\)，则C是NPC

    Cook-Levin引理：

    \(\forall A \in NP, A \leq_P SAT\)。即SAT是NP完全的

    过程：利用布尔串对应到布尔公式，利用接受的形式定义

    任取\(A\in NP\)，设N是A的\(n^k\)时间NTM，\(\forall w(|w| = n), N\)接受w

    等价于 N有对w长度小于\(n^k\)的接受格局序列。等价于能填好N在w上的画面（一个\(n^k\times n^k\)表格。等价于\(\phi = f(w)\)可满足（\(|\phi| = O(n^{2k})\)）

    本质上是将输入w的接受格局序列编码成布尔公式。推论3SAT也是NPC（将1个k-文字子句转化为k-2个3-文字子句）

    具体：

    构造\(\phi = f(w) = \phi_{cell} \land \phi_{start} \land \phi_{accept} \land \phi_{move}\)

    • \(\phi_{cell}\)：保证每个格子不为空，保证至少一个符号填在里面
    • \(\phi_{start}\)：规定起始状态（表第一行）
    • \(\phi_{accept}\)：保证至少有一个格子出现接受状态
    • \(\phi_{move}\)：保证格局序列演化正确

    等价关系：\(w\in A \Leftrightarrow <\phi> \in SAT\)。

    • 左到右：如果原来的w能被接受，则对应接受路径填出的表，能让\(\phi\)赋值为真
    • 右到左：如果\(<\phi>\)可满足，则从赋值反推计算表，由计算表可得接受w的一条完整格局序列

    即所有NP问题都能规约到SAT上。则只需要证明\(f(w) = <\phi>\)是多项式规约。表格大小为\(n^k\times n^k\)，且每个子公式长度都控制在\(O(n^{2k})\)内。因为k是图灵机运行时间的指数，由问题A决定，则\(|<\phi>|\)是关于输入长度n的多项式。

    证明3SAT是NPC

    只需要证明\(SAT \leq_P 3SAT\)

    拆\(\phi\)中四个子句：

    \[a_1 \lor a_2 \lor \dots \lor a_k\]
    \[(a_1\lor a_2 \lor \dots \lor a_{k-2} \lor z) \land (\neg z \lor a_{k-1} \lor a_k)\]

    再反复拆前面那一项，可以得到k-2条3文字子句

    证明有向图哈密顿路径问题HP是NPC

    HP显然属于NP（给定路径，沿着走一遍即可验证）

    给定3cnf公式\(\phi\)，有n个变量\(x_1, \dots, x_n\)与k个子句\(c_1, \dots, c_k\)

    每个变量x对应一个钻石构件（看图）

    子句与钻石中水平部分的节点相连（看ppt）。普通变量从左到右，带非的从右到左...

    哈密顿路径对应在此图中，只能经过每个子句顶点有且只有一次，即正规路径

    • 路径从s出发，进入\(x_1\)的钻石，在每个钻石中，要么从左到右（代表变量1），要么从右到左
    • 每穿过一个钻石，路径可能会绕路去访问一个或几个子句顶点（只要有这个变量对应文字，且方向匹配）
    • 最后从\(x_n\)的钻石出来，到终点t

    可满足赋值对应正规路径

    证明：无向图哈密顿路径问题UHP是NPC

    即证明\(HP \leq_P UHP\)

    对于有向图的一个节点，变为三个节点，其中左边节点全连入的边，右边节点全连出的边。则可以进行规约

    UHP是NP：构造多项式时间内判断UHC的非确定图灵机，枚举所有节点的排列，看是不是路径，是则接受，否则拒绝。则UHC属于NP

    则UHP是NPC的

    证无向图哈密顿回路问题UHC是NPC

    加一个节点，a，让a与s、t相连，则可以规约

    证明TSP是NPC

    仍然可以用类似方法枚举，最后条件加权重和小于b，得到TSP是NP的

    把哈密顿回路问题的图，变为完全图，设定权重函数，如果新的边是原图的边，权重为1，否则为2.设定预算b=n

    证明0-1背包问题KS是NPC

    证明是NP：给定子集，检查是否等于目标数t，复杂度为O(n)，符合

    证明\(3SAT \leq_P KS\)，即把3SAT化为0-1背包问题

    构造\(\phi\)的编码

    构造数集\(A=\{y_1, \dots, y_n, z_1 , \dots, z_n, g_1, \dots, g_k, h_1 \dots, h_k\}\)。A中数全是n+k位十进制数，且满足：

    • 高n位：分别对应变量\(x_1, \dots, x_n\)，每位只能是0，1
    • 低k位：分别对应子句\(c_1, \dots ,c_k\)

    具体如下：

    构造目标数t，高的n位全是1，低k位全是3

    然后对每个变量\(x_i\)，构造两个数字\(y_i, z_i\)（对应x取1或0），在变量对应列写1（y，z都这样），在子句列，若该赋值能让某个子句恒为真，就在那一列写1

    构造填充数字。对每个子句\(c_j\)，构造两个填充数\(g_i,h_i\)。只在对应子句列写1，其余全为0

    然后进行模拟可满足赋值：例如如果\(x_1 = 0\)，就从A中选\(z_1\)，否则选\(y_1\)，依此类推。然后最后再加上填充数\(g_i, h_i\)，进行十进制加法一定能凑够t。

    对不能满足的赋值，不管怎么加\(g_i, h_i\)，一定不能凑到t

    则将3SAT规约到KS
