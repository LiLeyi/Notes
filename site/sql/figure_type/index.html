    数据类型

    数字类型

    • 严格数字类型：int,small int,decimal,numeric
    • 近似数字类型：float,real,double precision
    • 拓展类型：tinyint,medium int,bigint

    整数类型

    • tinyint(M)：字节1，字符长度4（最大显示宽度，例如此处有符号-128~127，长度共三位，加符号位，一共四位，含符号位）
    • smallint(M)：字节2，字符长度5
    • mediumint(M)：字节3，字符长度9
    • int(M)：字节4，字符长度11
    • bigint(M)：字节8，字符长度20

    超出类型范围的操作，会发生out of range错误。指定zerofill则自动添加unsigned

    小数类型

    浮点数

    • float(M,D)：4字节，单精度浮点数，M为总位数，D为小数位数，M，D可省略，默认为10，2
    • double(M,D)：8字节，双精度浮点数，M，D可省略，默认为16，4

    定点数

    • decimal(M,D)：M+2字节

    位类型

    • bit(M)：1-64，默认1，位字段类型
    • binary(M)
    • varbinary(M):类似char与varchar
    • tinyblob(M):L+1
    • blob(M):L+2
    • mediumblob(M):L+3
    • longblob(M):L+4

    select bin(列名),hex(列名) from 表名; 分别以二进制，十六进制显示数据

    日期、时间类型

    • date:4字节，值范围1000-01-01~9999-12-31
    • time(fsp):3字节，-838:59:59~838:59:59
    • datetime(fsp):8字节，-1000-01-01 00:00:00~9999-12-31 23:59:59（不会转换时区）
    • timestamp(fsp):4字节，19700101080001~2038年某一时刻（会自动转换时区）
    • year:1字节，1901-2155

    字符串类型

    • char(M)：定长，M为长度，M可省略，默认为1，M为字节数，0-255，存储长度跟字符集有关
    • varchar(M)：变长，M为最大长度，M可省略，0-65535，M为字节数，0-255，存储长度跟字符集有关
    • tinytext：允许长度0-255字节，值长度+2字节
    • text：允许长度0-65535字节，值长度+2字节
    • mediumtext：允许长度0-16777215字节，值长度+3字节
    • longtext：允许长度0-4294967295字节，值长度+4字节

    使用length(列名)可获取字符串长度，使用concat(列名,字符串)将字符串拼接到后面

    枚举类型

    enum(值1,值2...)

    插入值时可以写值，也可以写索引，索引从1开始，索引越界会报错。一次只能选择一个值

    set类型

    set(值1,值2...)

    一次可选择多个值

    插入多个值时：values中写'值1,值2...'，不能空格

    SELECT u.id AS user_id, r.id AS right_id FROM user u JOIN userroletable urt ON u.id = urt.user_id JOIN role ro ON urt.role_id = ro.id JOIN rightroletable rrt ON ro.id = rrt.role_id JOIN rights r ON rrt.right_id = r.id;
