    说明

    语句以“;”结尾，“->”链接

    查看

    • show databases |查看所有数据库
    • show errors|warnings |查看错误，警告
    • show engines |查看引擎
    • show grants |显示当前用户信息

    • show status |显示服务器状态信息

    • show tables |查看所有表

    • desc tables |查看单个表

    • use 数据库名 |进入数据库

    创建

    create table 表名( 列名 数据类型 限制, ... )

    删除

    • drop databases 数据库名 |删除数据库
    • drop table 表名 |删除表

    约束

    • primary key 主键
    • not null 不空
    • unique 不可重复
    • binary 二进制
    • unsigned 无符号
    • zerofill 用0填充
    • auto_increment 自动生成一列作为主键
    • default 值。设置默认值

    添加删除主键

    在声明时，constrain中写primary key
    在声明列名时，可单独在创建的函数中使用primary key(key1,key2,...)声明主键
    alter table 表名 add|drop primary key(列1,列2,...)

    插入删除值

    insert into 表名 values(...)

    delete from 表名 where 条件

    两表建立关系

    • 两个存在关系的表储存引擎必须相同
    • 两个存在关系的表数据类型必须相同

    create table 表名(
    ....,
    foreign key(列名,列名...) references 主表名字(主键名字) 外键设置
    )

    或者用 constraint 约束名字 foreign key(列名,列名...) references 主表名字(主键名字) 外键选项，来自己定义约束名字，否则需要先查询，才能进行删除等操作。外键约束名，可以通过show create table 表名查看，外键约束名是CONSTRAINT后的值。若找不到约束名，则无法将两个表解除关系

    想删除主表数据，需要先删除从表里所有数据，或者删除外键

    删除主表的外键：

    alter table 表名 drop foreign key 外键约束名

    增加外键：

    alter table 表名 add [constraint 约束名] foreign key(列名) references 从表名(主键)

    中括号内不一定要写

    外键设置：

    • on delete 和 on update
    • 属性：
    • set null 主表数据删除或更新时，从表对应外键值设为null
    • cascade （级联）主表数据删除或更新，从表对应数据也删除或更新
    • restrict （约束）从表有数据则主表不允许删除和更新
    • no action同restrict innodb 拒绝删除或更新

    例：on delete set null主表的值删除时将从表的对应数据设定为null
