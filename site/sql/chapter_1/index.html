    基本概念与介绍

    数据

    • 对客观事物的符号表示
    • 是数据库中存储的基本对象
    • 数据具有一定的格式（语法）和含义（语义）
    • 与其语义密不可分

    数据管理技术

    1. 人工管理阶段，20世纪五十年代之前
    2. 文件系统阶段，20世纪五十年代后期到20世纪六十年代
    3. 数据库系统阶段，20世纪六十年代后期以来

    数据库

    提供存储空间，存储数据的容器，可包含多个文件。特征：

    • 按一定数据模型组织，描述和储存
    • 数据共享
    • 减少冗余
    • 数据独立性高
    • 统一的数据控制功能

    数据库管理系统（DBMS）

    • 数据定义（DDL）
    • 数据操纵（DML）
    • 数据控制（DCL）
    • 数据库的运行管理

    数据库应用程序

    • 满足对数据管理的更高要求
    • 使数据管理更直观友好
    • 负责与DBMS通信

    其他

    • 数据库系统（DBS）
    • ODBC（Open Database Connectivity）
    • JDBC（Java Database Connectivity）
    • ADO.NET（ActiveX Data Objects.NET）
    • PDO（PHP Data Object）
    • node

    数据模型

    • 层次模型（例如学校，学院，系的关系）
    • 网状模型（每个数据用一个节点表示，每个节点与其它节点都联系）
    • 关系模型（指表内部的关系，主流）

    关系数据库的产生

    1. 1970 理论提出
    2. 1985 12条规则
    3. 1990 330条规则

    关系模式

    表

    某种特定类型数据的结构化清单，做到一个清单一个表，越类聚越好，可节省空间 分类

    • 实体表 真正的表格
    • 关系表 让两个表建立关系的表

    组成

    • 行
    • 列

    主键

    • 不一定只有一列，只要能用于确定一行的，都可以是主键
    • 唯一标识一行
    • 每个表最好都定义主键
    • 定义条件：非空，唯一
    • 一般不更新

    外键

    • 包含另一个表的主键值
    • 定义了两个表的关系

    概念结构设计

    定义

    从分析用户的数据对象及数据对象之间的联系出发，到获取E-R图的过程

    E-R图

    • 实体：每一类对象的集合
    • 属性：每一个实体集涉及的信息项
    • 联系
    • 一对一：A中一个实体最多与B中一个实体相联系
    • 一对多：A中一个实体可与B中多个实体相联系，通常将“一”的主键放到“多”中
    • 多对多：...。必须建立关系表

    设计策略

    • 自底向上
    • 自顶向下
    • 混合设计
